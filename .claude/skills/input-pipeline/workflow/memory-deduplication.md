# 向量检索与相似度去重核心逻辑

本文档定义了记忆写入时的去重与更新逻辑。

> **说明**：本节仅描述写入阶段的去重逻辑。检索阶段的相似度计算与 Rerank 详见 `retrieval-pipeline` skill。

---

## 1. 写入时去重流程

```
新输入 → LLM 分类判断 → 获取该分类下所有 Resources → 向量相似度匹配
                                ↓
          ┌─────────────────────┼─────────────────────┐
          ↓                     ↓                     ↓
     相似度 ≥ skip阈值      merge阈值 - skip阈值   < merge阈值
     （高度重复）          （相关但不重复）      （无关内容）
          ↓                     ↓                     ↓
     跳过存储               追加或新建            正常新增
     强化已有
```

---

## 2. 相似度阈值（分类特定）

不同分类采用不同的去重阈值，以适应各分类的特性：

### 2.1 Category 级别阈值

| 分类 | skip_threshold | merge_threshold | 说明 |
|------|----------------|-----------------|------|
| 核心自我 | 0.85 | 0.75 | 身份信息相对稳定，适度宽松 |
| 情景时间轴 | 0.90 | 0.75 | 情景细节需要更高相似度才跳过 |
| 语义知识库 | 0.88 | 0.75 | 知识信息需要较高的相似度 |
| 社交关系图谱 | 0.85 | 0.75 | 关系信息相对稳定 |
| 动态领域分类 | 0.92 | 0.75 | 动态分类需要非常高的相似度才跳过 |

### 2.2 Resource 级别阈值

| 阈值类型 | 值 | 说明 |
|----------|-----|------|
| skip_threshold | 0.90 | Resource 是综合摘要，需要高相似度才跳过 |
| merge_threshold | 0.80 | 合并阈值相对较高 |

### 2.3 默认阈值

对于未定义的分类，使用默认阈值：

| 阈值类型 | 值 |
|----------|-----|
| skip_threshold | 0.88 |
| merge_threshold | 0.75 |

---

## 3. 去重操作类型

| 操作类型 | 检测条件 | 示例 |
|----------|----------|------|
| **SKIP（跳过）** | 相似度 ≥ skip_threshold | 用户说"我爱喝咖啡"，已有"我喜欢喝咖啡" → 跳过，importance_score +1 |
| **MERGE（合并）** | merge_threshold ≤ 相似度 < skip_threshold，LLM 判断合并 | 已有："喜欢咖啡" → 新："每天早上喝美式" → 结果："喜欢咖啡，每天早上喝美式" |
| **UPDATE（更新）** | merge_threshold ≤ 相似度 < skip_threshold，LLM 判断更新 | 已有："25岁" → 新："其实26岁了" → 结果："26岁" |
| **CREATE（新建）** | 相似度 < merge_threshold 或 LLM 判断独立 | 已有："学Python" → 新："学机器学习" → 两条独立记录 |

---

## 4. 相似度冲突解决策略

当相似度在 **merge_threshold - skip_threshold** 之间时，使用 LLM 进行智能判断：

### 4.1 判断 Prompt 模板

```
# 已有记忆
[已有内容]

# 新输入
[当前输入]

# 任务
判断新输入与已有记忆的关系，并返回操作类型：

1. **merge**：新信息是对已有记忆的补充，应合并到已有记忆中
2. **update**：新信息修正/覆盖已有记忆
3. **create**：新信息是独立内容，应新建记录

# 输出格式
仅输出 JSON，不要包含任何解释：
{
  "action": "merge" | "update" | "create",
  "reason": "判断理由",
  "merged_content": "合并/更新后的内容（仅 merge/update 时需要）"
}
```

### 4.2 判断逻辑

- **merge**：新信息扩展了已有记忆的细节
- **update**：新信息修正了已有记忆的错误
- **create**：主题相似但属于不同时间/场景的独立事件

---

## 5. 强化机制

当执行 **SKIP** 操作时，对已有记忆进行强化：

```python
# 强化已有记忆
new_importance_score = min(existing.importance_score + 1, 10)
```

这样可以确保高频出现的记忆（即使内容相似）获得更高的重要性分数。
