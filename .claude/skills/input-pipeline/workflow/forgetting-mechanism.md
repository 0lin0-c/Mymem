# 艾宾浩斯遗忘曲线与访问加成逻辑

本文档定义记忆遗忘机制的实现规范。

---

## 1. 固定分类保护

以下分类**不参与遗忘机制**，只增不改：

- **核心自我**：身份信息、偏好习惯是用户的核心特征，不应遗忘
- **社交关系图谱**：人际关系是长期记忆，不应遗忘

对于这两个分类，只支持：
- 新增信息（如新爱好、新朋友）
- 更新信息（如爱好变化、联系方式变更）

---

## 2. 遗忘曲线模型

采用 **艾宾浩斯遗忘曲线** 的改进版本，引入"访问加成"因子：

```
EffectiveScore = Importance × e^(-Days / (Importance × 5)) × (1 + log(AccessCount + 1))
```

**公式拆解**：

| 因子 | 公式 | 说明 |
|------|------|------|
| 基础重要性 | `Importance` | 原始重要性评分 (0-3) |
| 时间衰减 | `e^(-Days / (Importance × 5))` | 半衰期与重要性正相关 |
| 访问加成 | `(1 + log(AccessCount + 1))` | 对数增长，防止过度膨胀 |

**参数说明**：
- `Days`：距创建/最后更新的天数
- `AccessCount`：被检索引用的次数（`access_count` 字段）
- `Importance × 5`：动态半衰期，重要性越高衰减越慢

---

## 3. 衰减效果示例

| 原始分数 | 访问次数 | 30天后 | 90天后 | 180天后 |
|----------|----------|--------|--------|---------|
| 10 | 0 | 9.7 | 9.1 | 8.2 |
| 5 | 0 | 3.7 | 2.0 | 0.9 |
| 5 | 10 | 5.3 | 2.9 | 1.3 |
| 3 | 50 | 4.1 | 2.2 | 1.0 |

> **设计意图**：即使是随口提的小事，如果被 AI 反复引用，也会变成"长期记忆"。访问次数越多，记忆越难被遗忘。

---

## 4. 遗忘触发条件

| 条件 | 操作 |
|------|------|
| 有效重要性 < 1 | 标记为"待遗忘"（importance_score = 1） |
| 有效重要性 < 0.5 且超过 min_days 天未访问 | 彻底删除 |
| 用户主动删除 | 立即删除 |

**默认参数**：
- `threshold = 0.5`：删除阈值
- `min_days = 30`：最小未访问天数

---

## 5. 重要性衰减流程

`MemoryLifecycle.decay_importance()` 执行流程：

```
1. 获取用户所有 Resource 和 Category
2. 对每条记忆计算有效重要性
3. 如果有效重要性 < 原始重要性 - 0.5，更新 importance_score
4. 跳过核心自我和社交关系图谱
```

---

## 6. 遗忘清理流程

`MemoryLifecycle.forget_low_importance()` 执行流程：

```
1. 获取用户所有 Resource 和 Category
2. 对每条记忆计算有效重要性
3. 检查距最后更新的天数
4. 满足删除条件（effective_importance < threshold 且 days > min_days）则删除
5. 跳过核心自我和社交关系图谱
```

---

## 7. 与检索评分的区别

| 对比项 | 检索评分（四因子） | 遗忘评分 |
|--------|-------------------|----------|
| **目的** | 排序检索结果 | 判断是否删除记忆 |
| **时间衰减** | `exp(-0.693 × days / 60)` 固定半衰期 | `exp(-days / (importance × 5))` 动态半衰期 |
| **相似度** | `cosine_similarity` | 无（遗忘不需要语义匹配） |
| **触发时机** | 每次检索时 | 定期清理或手动触发 |
