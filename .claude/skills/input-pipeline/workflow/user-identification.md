# 身份识别与拦截逻辑

本文档定义了用户识别机制和未识别时的拦截逻辑。

---

## 1. 识别流程

```
用户发送请求
      ↓
系统提取 Metadata (JWT / DeviceID / SessionID)
      ↓
┌─────────────────────────────────────────────────────┐
│ 未绑定 UserID：                                      │
│   1. LLM 引导："初次见面，请问怎么称呼您？"           │
│   2. 匹配数据库：已有用户直接识别，新用户自动创建      │
│   3. 成功后：下发加密 Cookie/Token，后续请求静默识别  │
└─────────────────────────────────────────────────────┘
      ↓
已识别：直接使用 user_id 处理请求
```

---

## 2. 身份识别机制

| 场景 | 识别方式 | 说明 |
|------|---------|------|
| **有前端登录** | JWT/Session | 用户登录后，从 Token 获取 user_id，直接绑定 |
| **无前端/CLI** | LLM 对话识别 | 用户输入名字直接识别，匹配已有用户或创建新用户 |

---

## 3. 实现阶段

| 阶段 | 识别方式 | 说明 |
|------|---------|------|
| **当前（无前端）** | LLM 对话识别 | 每次新会话，LLM 主动询问用户身份 |
| **后续（有前端）** | 登录态自动识别 | 从 JWT Token / Session 获取 user_id |
