# 多模态处理规则

本文档定义了不同模态（modality）的处理策略。

---

## 1. 模态类型

| modality | 说明 | 存储策略 | 状态 |
|----------|------|---------|------|
| `text` | 文本对话 | 每 5 轮批量存储 | ✅ 已实现 |
| `image` | 图片/截图 | 立即存储 | 🔜 骨架已搭建 |
| `video` | 视频 | 立即存储 | 🔜 骨架已搭建 |
| `voice` | 语音 | 立即存储 | 🔜 骨架已搭建 |
| `document` | 长文档 | 立即存储 | 🔜 骨架已搭建 |

---

## 2. 存储策略

### 2.1 chat 模态（text）

- 每 5 轮批量存一次数据库（减少 API 调用和数据库写入）
- 详见 `workflow/chat-caching.md`

### 2.2 其他模态（image/video/voice/document）

- 每次立即存数据库
- chat 计数不受其他模态影响，继续累积

---

## 3. 跨模态保护

```
用户发送 chat（第1轮） → 缓存，不存数据库
用户发送 chat（第2轮） → 缓存，不存数据库
用户发送 image       → 立即存数据库（chat 计数不变）
用户发送 chat（第3轮） → 缓存，不存数据库
用户发送 chat（第4轮） → 缓存，不存数据库
用户发送 chat（第5轮） → 批量存数据库，清空缓存，计数重置
```
