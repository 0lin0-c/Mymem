# LLM 充足性判断

本文档定义检索流程的第三步：判断 Category 层检索结果是否足够回答用户问题。

---

## 1. 设计目的

避免不必要的 Resource 层检索，提升响应速度。

| 场景 | 判断结果 | 后续动作 |
|------|----------|----------|
| Category 结果语义完整、信息充分 | 足够 | 直接构建上下文回答 |
| Category 结果模糊、缺少细节 | 不足 | 进入 Resource 层检索 |
| 无 Category 结果 | 不足 | 直接进入 Resource 层检索 |

---

## 2. 判断 Prompt 要点

让 LLM 根据以下信息判断：
- 用户问题
- 检索到的 Category 记忆片段

输出 JSON 格式：
```json
{
    "sufficient": true/false,
    "reason": "简要说明判断理由"
}
```

---

## 3. 判断规则

1. 记忆片段提供了回答问题所需的全部关键信息 → 足够
2. 记忆片段信息模糊、缺少细节、需要更多上下文 → 不足
3. 优先判断为"足够"，避免过度检索

---

## 4. 降级策略

LLM 调用失败时：
- 四因子评分 >= 0.6 → 足够
- Category 结果 >= 3 条 → 足够
- 其他情况 → 进入 Resource 层

> **注意**：降级策略使用四因子评分而非纯余弦相似度，因为检索结果已包含四因子评分。
