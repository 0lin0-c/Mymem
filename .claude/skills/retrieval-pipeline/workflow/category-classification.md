# LLM 分类判断

本文档定义检索流程的第一步：通过 LLM 判断用户查询应检索哪些分类。

---

## 1. 设计原则

**动态数量**：LLM 根据问题复杂度决定类别数量，而非固定值。

| 问题类型 | 类别数量 | 示例 |
|----------|----------|------|
| 单一明确 | 1 个 | "我的项目进度" → [项目研发] |
| 多维度 | 2-3 个 | "我最近学习情况怎么样" → [核心自我, 考试与升学] |
| 开放式 | 可能多个 | "帮我回顾一下最近的事" → [情景时间轴, 核心自我] |

---

## 2. 分类判断 Prompt

```
# Role
你是一个记忆分类专家。根据用户查询，判断需要检索哪些类别的记忆。

# 用户查询
{user_query}

# 可用分类
{available_categories}

# 判断规则
1. 只返回与查询直接相关的分类
2. 数量不限，但不要过度泛化
3. 如果查询明确指向某个领域，只返回该领域
4. 如果查询模糊，可以返回多个相关分类

# 输出格式
返回 JSON 数组：
["分类名1", "分类名2", ...]
```

---

## 3. 返回示例

**查询**："我那个 Python 项目进度怎么样了？"

**LLM 返回**：
```json
["项目研发"]
```

**查询**："最近有什么重要的事情需要我处理吗？"

**LLM 返回**：
```json
["情景时间轴", "项目研发"]
```
