# 核心架构与设计哲学

本文档定义数据库层的设计原则和核心理念。

---

## 1. 全异步 I/O (asyncpg)

AI Agent 系统的主要瓶颈在于大模型 API 请求和向量数据库检索。强制在底层引擎层采用全异步驱动，确保在等待数据库返回几万条高维向量比对结果时，主线程不会阻塞。

---

## 2. 三层记忆结构

### 2.1 Resource 表（对话摘要层）

存储对话粒度的综合摘要与 1536 维向量。追求**语义检索能力**。

### 2.2 Category 表（原子化记忆层）

存储从对话中提取的原子化信息。每条记录属于一个分类（核心自我/情景时间轴/语义知识库/社交关系图谱/动态分类）。

### 2.3 ResourceCategory 表（关联层）

记录对话摘要与原子化记忆之间的关系，支持追踪每条记忆的完整历史来源。

---

## 3. UUID 主键约束

所有表放弃使用自增 Integer 作为主键，强制使用 UUIDv4。这在未来如果需要进行数据库分库分表，或者进行离线数据迁移时，能彻底杜绝主键冲突。
