# 用户画像表单

本文档定义用户画像初始化阶段的表单字段设计。

> **说明**：以下字段为前端初始化流程需要收集的信息，后端处理后统一整合存入数据库，不单独存为字段。
> - 用户名称存入 `username` 字段
> - 其余信息整合后存入 `user_prompt_template` 字段

---

## 1. 基本信息（必填）

| 字段 | 类型 | 必填 | 说明 | 校验规则 |
|------|------|------|------|----------|
| `username` | string | 是 | 用户名称 | 1-50 字符 |
| `identity_type` | enum | 是 | 身份类型 | 见下方枚举 |

### identity_type 枚举值

| 提交值 | 中文显示 |
|--------|----------|
| `student` | 学生 |
| `worker` | 上班族 |
| `teacher` | 教师 |
| `freelancer` | 自由职业者 |
| `other` | 其他 |

---

## 2. 身份详情（根据 `identity_type` 动态展示）

> **字段结构**：`identity_detail` 为对象，内含多个可选字段。前端根据 `identity_type` 决定展示哪些字段。

### 2.1 学生 (student)

| 字段 | 类型 | 必填 | 说明 | 提交值 |
|------|------|------|------|--------|
| `education_stage` | enum | 否 | 学段 | 见下表 |
| `major` | string | 否 | 专业方向 | 自由文本 |

**education_stage 枚举值：**

| 提交值 | 中文显示 |
|--------|----------|
| `elementary` | 小学 |
| `middle` | 初中 |
| `high` | 高中 |
| `college` | 大学 |
| `graduate` | 研究生 |

### 2.2 上班族 (worker)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `industry` | string | 否 | 所在行业 |
| `job_title` | string | 否 | 职位名称 |

### 2.3 教师 (teacher)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `subject` | string | 否 | 任教学科 |
| `teaching_stage` | string | 否 | 教学学段 |

### 2.4 自由职业者 (freelancer)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `field` | string | 否 | 从事领域 |

### 2.5 其他 (other)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `description` | string | 否 | 自定义身份描述 |

---

## 3. 使用场景（可选，多选）

> **字段**：`use_cases`，类型为 `string[]`

| 提交值 | 中文显示 | 说明 |
|--------|----------|------|
| `学习辅助` | 学习辅助 | 作业答疑、知识讲解、考试准备 |
| `工作助手` | 工作助手 | 文档处理、方案策划、技能提升 |
| `生活顾问` | 生活顾问 | 日程管理、健康建议、生活决策 |
| `情感陪伴` | 情感陪伴 | 倾诉交流、情绪支持 |
| `创意灵感` | 创意灵感 | 写作辅助、头脑风暴、创作建议 |

---

## 4. 兴趣标签（可选，多选）

> **字段**：`interests`，类型为 `string[]`，直接提交中文标签名

| 类别 | 标签选项 |
|------|---------|
| 科技 | 编程、AI、数码、科学 |
| 人文 | 文学、历史、哲学、心理学 |
| 艺术 | 音乐、绘画、电影、摄影 |
| 生活 | 美食、旅行、运动、游戏 |
| 发展 | 投资、职场、语言、技能 |
